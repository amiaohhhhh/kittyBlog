BOM-浏览器对象模型

# BOM 浏览器对象模型

# 概述

# 什么是BOM

BOM是浏览器对象模型，提供了独立于内容而与浏览器窗口进行交互的对象，其核心对象是window.
BOM由一系列相关的对象构成，并且每个对象都提供了很多方法于属性
但是BOM缺乏标准

# DOM与BOM

  • 从对象模型看，DOM是文档对象模型则把文档当作一个对象来看待，BOM是浏览器模型则把浏览器当作一个对象来看待，所以BOM包括DOM
  • 顶级对象看，DOM的顶级对象是document,BOM的顶级对象是wiindow
  • 学习内容看，DOM主要学习操作页面元素，BOM学的是浏览器窗口交互的一些对象
  • DOM的兼容性比BOM强

# BOM构成

window对象是浏览器的顶级对象

  • 它是js访问浏览器窗口的一个接口

  • 它是一个全局对象，定义在全局对象域中的变量，函数都会变成window对象的属性和方法

    在调用的时候可以省略window,前面学习的对话框都属于window对象方法，如alert(),prompt()

# 1.1.1 window.onload()

window.onload()是窗口(页面)加载事件，当文档内容完全加载完成会触发该事件(包括图像、脚本文件、css文件等)，就调用的处理函数

有了window.onload就可以把js代码写到页面元素的上方，因为onload等页面内容全部加载完毕，再去执行处理函数
传统注册事件方式只能写一次，如果有多个会以最后一个window.onload为准

# 案例--倒计时

    <span class="h"></span>
    <span class="m"></span>
    <span class="s"></span>
    <script>
        var hour = document.querySelector('.h');//获取元素
        var min = document.querySelector('.m');
        var second = document.querySelector('.s');
        var inputTime = +new Date('2021-5-1 18:00:00');//返回输入时间的毫秒数
        countDown();//先调用一次函数，防止第一次刷新页面有空白
        setInterval(countDown,1000);调用计时器，一秒钟调用一次
        function countDown(){
            var nowTime = +new Date();//获取当前时间的毫秒数
            var times = (inputTime - nowTime)/1000;
            var h = parseInt(times / 60 / 60 % 24);
            h = h < 10 ? '0' + h : h;
            hour.innerHTML = h;
            var m = parseInt(times / 60 % 60);
            m = m < 10 ? '0' + m : m;
            min.innerHTML = m;
            var s = parseInt(times % 60);
            s = s < 10 ? '0' + s : s;
            second.innerHTML = s;
        }
    </script>

        var btn = document.querySelector('button');
        var time = 3;
        btn.addEventListener('click',function(){
            btn.disabled = true;

# 2 JS运行机制

# 2.1 同步和异步

js本身是单线程

# 2.1.1 同步

前一个任务结束后再执行后一个任务，程序执行顺序与任务排列顺序是一致的、同步的。

# 2.1.2 异步

可同时处理多个事件
区别：各个流程的执行顺序不一样

# 2.1.3 同步任务

都在主线程上执行，形成一个执行线

# 2.1.4 异步任务

是通过回调函数实现的

  1. 普通事件 click、resize
  2. 资源加载 load、error
  3. 定时器 setlnterval、setTimeout

相关回调函数添加到任务队列中

# 2.1.5 执行机制

  1. 先执行执行栈中的同步任务
  2. 异步任务(回调函数)放入任务队列中
  3. 一旦执行栈中的所有同步任务执行完毕，系统就会按次序读取任务队列中的异步任务，遇是就读取的异步任务结束等待状态，进入执行栈，开始执行

# 2.2 location对象

Location 对象包含有关当前 URL 的信息。
Location 对象是 Window 对象的一个部分，可通过 window.location 属性来访问。
对象方法

  • location.assign() 跟href()一样，可以跳转页面
  • location.replace()    替换当前页面，因为不记录历史，所以不能后退页面
  • location.reload()    重新加载页面，相当于刷新按钮或者f5如果参数为true强制刷新ctrl+f5

# 3 pc端网页特效

# 3.1 元素偏移量offset系列

# 3.1.1 概述

可以动态得到盖元素的位置(偏移)，大小等

  • 获取元素距离带有定位父元素的位置
  • 获得元素自身的大小(宽度高度)
  • 注意：返回的数值不带单位 | element.offsetParent | 返回该元素带有定位的父集元素，没有定位返回body | | | | --- | --- | --- | --- | | element.offsetTop | 定位父元素上方的偏移 | | | | element.offsetLeft | 定位父元素左边框的偏移 | | | | element.offsetWidth | 自身包括padding、边框、内容区的宽度，数值不带单位 | | | | element.offsetHeight | 自身包括padding、边框、内容区的高度，数值不带单位 | | |

# 3.1.2 offset与style的区别

offset

  • offset可以得到任意样式表中的样式值

style只能得到行内样式中的样式值

  • 获得的数值是没有单位的

style.width获得的是带单位的字符串

  • offsetWidth包含Padding+border+width

style.width获得不包含padding和border的值

  • offsetWidth等属性是只读属性，只能获取不能赋值

style.width是可读属性，可以获取也可以赋值

  • 当获取元素大小位置用offset更合适

想给元素更改值，需要用style改变

# 3.2 元素可视区client系列

使用该系列相关的属性来获取元素可视区的相关信息

element.clientTop 返回元素上边框的大小
element.clientLeft 元素左边框大小
element.clientWidth 返回自身包括padding\内容区的宽度，不含边框，返回数值不带单位
element.clientHeight 返回自身包括padding\内容区的高度，不含边框，返回数值不带单位

# 3.3 元素滚动scroll系列

得到元素实际大小，可以动态得到该元素的大小，滚动距离等
注意是被卷去，即是内容超出盒子的距离

element.scrollTop 返回被卷去的上侧距离，返回数值不带单位
element.scrollLeft 返回被卷去的左侧距离，返回数值不带单位
element.scrollWidth 返回自身实际宽度，不含边框，返回数值不带单位
element.scrollHeight 返回自身实际高度，不含边框，返回数值不带单位

element.scrollTop元素卷去的头部
window.pageYOffset页面被卷去的头部
window.pageXOffset页面被卷去的左侧

# 页面被卷去的头部兼容性解决方案

声明了<!DOCTYPE html> 使用documentElement.scrollTop
没有声明 document.body.scrollTop
新方法 window.pageYOffset和window.pageXOffset,ie9支持
封装一个方法

function getScroll(){
	return{
  	left:window.pageXOffset || document.documentElement.scrollLeft || document.body.scrollLeft || 0
    top:window.pageYOffset || document.documentElement.scrollTop || document.body.scrollTop || 0
  }
}

# 3.4 mouseenter和mouseover的区别

mouseenter鼠标事件

  • 当鼠标移动到元素上时就会触发mouseenter事件
  • 类似mouseover，它们两者之间的差别
  • mouseover鼠标经过自身盒子会触发，经过子盒子还会触发，mouseenter只会经过自身盒子触发 因为会冒泡
  • 因为mouseenter不会冒泡
  • 跟mouseenter搭配鼠标离开mouseleave同样不会冒泡
