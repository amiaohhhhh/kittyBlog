# 评论模块

本站使用过Gitalk、valine，目前使用valine评论系统

# 前言

  • Gitalk 基于 GitHub 的 Issue
  • Valine 基于 LeanCloud

Gitalk、Valine 通过使用 vuepress-plugin-comment (opens new window)进行搭建。它既支持 Gitalk 应用也支持 Valine。

# Gitalk

如果项目部署在了GitHub中，直接评论绑定了博客，评论后信息会存储在博客的Issues里面

# 优点

  • 使用 GitHub 登录(所以你要有一个GitHub账号)
  • 支持多语言 [en, zh-CN, zh-TW, es-ES, fr, ru]
  • 支持个人或组织
  • 无干扰模式（设置 distractionFreeMode 为 true 开启）
  • 快捷键提交评论 （cmd|ctrl + enter）

缺点：

  • 不能自定义名字、头像发表评论
  • 没有自定义表情

# 开始部署

  1. 先关闭其他评论系统的插件
  2. 安装（2选一）
yarn add vuepress-plugin-comment -D
1
npm install vuepress-plugin-comment -D
1

3.申请权限 点击申请 (opens new window)
参数说明Application name：应用名称
Homepage URL：应用程序主页完整URL
Application description：应用说明 可选
Authorization callback URL：应用程序的回调 URL
申请后会有个Client ID

到.vuepress/config.ts中 // 插件配置

module.exports = {
  plugins: [
    [
      'vuepress-plugin-comment',
      {
        choosen: 'gitalk', 
        options: {
          clientID: '你的client ID', 
          clientSecret: '你的client secret',  // 你的client secret
           repo: 'GitHub 仓库', // GitHub 仓库
            owner: 'amiaohhhhh', // GitHub仓库所有者
            admin: ['amiaohhhhh'], // 对仓库有写权限的人
            // distractionFreeMode: true, 无干扰模式
            pagerDirection: 'last', // 'first'正序 | 'last'倒序
            // 下面的不用修改，默认这些即可
            id: '<%- (frontmatter.permalink || frontmatter.to.path).slice(-16) %>', //  页面的唯一标识,长度不能超过50
            title: '「评论」<%- frontmatter.title %>', // GitHub issue 的标题
            labels: ['Gitalk', 'Comment'], // GitHub issue 的标签
            body:
              '页面：<%- window.location.origin + (frontmatter.to.path || window.location.pathname) %>', // GitHub issue 的内容
        }
      }
    ]
  ]
}

# 问题

1.授权后出现Error:Network error
网络太卡了....(这也是我换的原因)
2.评论区出现 Error: Validation Failed.
可能页面的链接过长，超过 50，请求 Issues 失败。
解决：手动设置 id 取值，限制长度不超过 50
3.切换页面后评论区内容还是上一个页面的评论
frontmatter.to.path 没有即时获取 path
2和3问题的解决

已经加在前面的配置上了，解释一下

{
  choosen: 'gitalk', 
    options: {
    ...
    id: "<%- (window.location.origin + (frontmatter.to.path || window.location.pathname)).slice(-50) %>", //  页面的唯一标识,长度不能超过50
    title: "「评论」<%- document.title %>", // GitHub issue 的标题
    labels: ["Gitalk", "Comment"], // GitHub issue 的标签
    body:"<%- document.title %>：<%- window.location.origin + (frontmatter.to.path || window.location.pathname) %>" // GitHub issue 的内容
  }
}

# Valine

一款快速、简洁且高效的无后端评论系统。
官网 (opens new window)

# 优点

  • Valine 是基于 LeanCloud 构建，Gitalk 是基于 Github 的 Issue 构建
  • Valine 使用简单，可以自定义名字、头像后发表评论，无需登录任何账户，Gitalk 必须登录自己的 Github 账号才能发表评论
  • Valine 支持自定义 emoji，如 B 站，微博等的表情 ...

# 开始部署

1.关闭其他评论系统的插件
2.注册登录LeanCloud
进入控制台 (opens new window)后点击左下角创建应用 (opens new window)：

应用创建好以后，进入刚刚创建的应用，选择左下角的设置>应用Key，然后就能看到你的APP ID和APP Key了：

3.到配置文件.vuepress>config.ts中评论模块配置

module.exports = {
    plugins: [
        [
            'vuepress-plugin-comment',
            {
                // 具体参数请看官网：https://valine.js.org/
                choosen: 'valine',
                // options 选项中的所有参数，会传给 Valine 的配置
                options: {
                    el: '#valine-vuepress-comment',
                    appId: '你的 AppID',
                    appKey: '你的 AppKey',
                    placeholder: '请留下你的足迹 ~~',
                    // 有 URL、''、mp、identicon、monsterid、wavatar、retro、robohash、hide 头像选择，具体头像是什么样子，请访问 https://valine.js.org/avatar.html
                    avatar: 'mp',	// 默认头像
                    pageSize: 10,   // 评论列表分页，每页条数
                    visitor: true,    // 文章访问量统计
                    recordIP: false,   // 是否记录评论者 IP
                    enableQQ: true,   // 是否启用昵称框自动获取 QQ 昵称和 QQ 头像, 默认关闭
                }
            }
        ]
    ]
}
