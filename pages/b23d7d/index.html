# 《ES6标准入门》书摘

# 1. let和const命令

# 1.1 const

  • 声明的是常量，不能改变
  • 一声明就必须立即初始化，不能留到后面
  • 命令所在代码快内有效
  • 暂时性死区 不重新声明
  • 只是保证变量名指向地址不变，但是不保证地址的数据不变，所一个对象声明为变量必须非常小心
  • 冻结Object.freeze() 不让其他对象数组赋值给变量

# 1.2 let

  • 命令所在代码块内有效
  • 适合：for循环
  • 不存在变量提升：变量一定要在声明后使用
  • 暂时性死区(TDZ)
    • 只要块级作用域内存在let命令，就绑定这个区域，不受外部影响 代码块中在let变量没有申请前是不可用的
      • var temp = 1;  if(true){temp = ‘abc’; // referenceError  let temp;}
    • 本质：只要一进入当前作用域，所用变量a就已经存在且使用了，但是不可获取，只有等声明变量的那一行代码出现，才可以获取使用该变量a
    • 用于减少运行错误，防止在变量声明前就使用这变量
    • function  bar(x = y,y =2){return [x,y]}  bar() //报错
      • 因为y在x调用后才声明变量
  • 不允许重复声明
  • 跨模块常量
export const A = 1 //constants.is模块    
import * as constants from ‘./constants’ // test1.js模块   
console.log(constants.A) //1    
import {A} from ‘./constants’   
console.log(A) //1

# 2. 变量的解构赋值

# 2.1 数组

  • var a  = 1； var b=2;
    ES6允许写成 var [a,b]=[1,2] 模式匹配 若变量比对应数值多匹配不成功返undefined
  • 只要某种数据结构具有 Iteration接口，都可以采用数组形式解构赋值

  • 允许指定默认值
[x,y = 3] = [‘a’]  //x=‘a’ y = 3
1
  • 等号右边不是数组会报错
  • 如果一个数组成员不===undefined,默认值不生效
var [x = 1] = [ undefined] // x = 1
1
  • 如果一个数组成员是null，默认值就不会生效，因为null不===undefined
var [x = 1] = [null] // x= null
1
  • 如果默认值是一个表达式，那么这个表达式是惰性求值，只有在用的时候，才会求值
  • 默认值可以引用解构赋值的其他变量，但该变量必须已经声明
let [x = 1,y = x] = [] // x=1;y=1
1

# 2.2 对象

  • 与数组不同
    • 数组元素按次序排列，变量取值由它位置决定
    • 对象属性没有次序，变量必须与属性同名
var { a , b}={a:12,b:34} //a=12 b=34
1
  • 对象解构赋值内部机制
    • 先找到同名属性->赋给对应的变量
      • 真正被赋值是后者
var {a:b} = {a :12,b:66} 
b=66 
a:undefined
1
2
3
  • 对于let和const来说，变量不能重新声明，所以一旦赋值的变量以前声明过，就会报错。
let t ; 
let { y } = { y :1} // 报错
1
2
  • 如果没有第二 let命令，上面代码不会报错
let t; { y } = { y :1} // y = 1
1
  • 模式没值赋，可用于嵌套
  • 可指定默认值 ：对象的属性值严格等于undefined
var { x = 3} = {} x = 3
var {x=3} = {x:null}  //x = null
1
2
  • 错误的写法
    • 子对象所在的父属性不存在
var {x:{b}} = {b:34}  // 报错
1
  • 要非常小心已经声明的变量用于解构值
var x; {x}={x:1} //报错
//JS引擎会将{x}理解成一个代码块，从而发生语法错误
var x；({x} = {x=2})
