起步

# vuepress-theme-vdoing做个人博客

# 介绍

基于vuepress做的一个主题
vuepress-theme-vdoing官网 https://doc.xugaoyi.com/ (opens new window)
vuepress官网 https://www.vuepress.cn/ (opens new window)
‼️ 注意
建议先看一遍vuepress目录然后再去vuepress-theme-vdoing中进行初步操作

# 前期准备

gitHub账号
node版本大于14(我用的最新版，若版本过低的话需要卸载掉node再到官网安装)
yarn/npm

# 安装

1.选择自己想要发风格进行安装 我选的是知识库兼博客

# 拷贝项目

git clone https://github.com/xugaoyi/vuepress-theme-vdoing.git (opens new window) //如果不能下载成功的话 使用 git clone git@github.com:xugaoyi/vuepress-theme-vdoing.git

# cmd 运行

cd vuepress-theme-vdoing

# install dependency 注意：如安装不成功请关闭淘宝源。

npm install # or yarn install

# develop

npm run dev # or yarn dev

# 部署到gitHub page中

因为我还不懂服务器这些东西 怕被攻击 所以保守点直接放到github page里面
首先在github里加个空白的项目即可

# 1.在vuepress项目中改变自己的配置

打开deploy.sh 拷贝这段代码完全覆盖


#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# 写好自己的github账号 以及你准备这个项目的目录
git push -f git@github.com:<USERNAME>/<项目目录>.git master:gh-pages

cd -

# 2.添加base

:::info 目录地址:docs/.vuepress/config.ts ::: 如果你的github项目目录叫 blog 则 base:/blog/
‼️ 注意
如果你没有写好 会报错的哈 默认是/

# 3.修改命令

:::info 目录地址:package.json :::

"scripts":{
  //在这里面添加
  "deploy":'bash deploy.sh'
}

设置成功后 后面直接在git软件输入 'yarn deploy' 运行成功即可

# 最后

运行命令后到官方网站查看是否push成功
点击项目中的settIng查看配置"github pages"是否配置成功

# 接入到vercel

到官网 https://vercel.com/ (opens new window)
登录后 点击create ... 进入这个页面

‼️ 注意
用vuepress 其他默认即可 记得把这个打开

‼️ 注意
建好后 如果分支是gh-page 会因为vercel会选择默认的main
建完后 需要到 建好项目中的设置进行调整 然后等到下一次push时才生效

# 第三方软件

瓦雀 :::info https://github.com/yesmeck/waque.git (opens new window) ::: 使用教程 github都写了
可用于语雀导出到本地 命令行 waque export [知识库名]

# 1.没有自动生成目录

想要点击关于后会弹到目录页面 侧边框


解决：

在00.目录页中配置所需要的生成的目录

.vuepress>config.ts 记得写好文件名前序号，不然不会生成
