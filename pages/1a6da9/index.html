# webpack中的插件

**2.html-webpack-plugin **复制到项目根目录中的index.html页面，也被放到内存中 在生成的index.html页面，自动注入了打包的bundle.js文件

配置：

//1.导入HTML插件，得到一个构造函数
const HtmlPlugin = require('html-webpack-plugin')

//2.创建HTML插件的实例对象
const htmlPlugin = new HtmlPlugin({
  template:'./src/index.html',//指定原文件的存放路径
  filename:'./index.html',//指定生成的文件的存放路径
})
module.exports = {
  mode:'development',
  plugins:[htmlPlugin],//通过plugins节点，使htmlPlugin插件生效
}

# 1.概述

在实际开发过程中,webpack默认只能打包处理以.js后缀名结尾的模块.非.js后缀名结尾的模块,默认不处理,需要调用loader加载器才正常打包,否则报错

# 打包处理css文件

运行npm i style-loader@3.0.0 css-loader@5.2.6 -D 命令,安装处理css文件中的loader
当webpack发现某个文件处理不了的时候,会查找webpack.config.js这个配置文件,看module.rules数组中,是否配置了对应的loader加载器

module:{
  rules:[
    //定义了不同模块对应的loader
    {test:/\.css$/,use:['style-loader','css-loader']}
  ]
}

1
2

优：提高性能

# 打包处理样式表中与url路径相关的文件

运行npm i url-loader@4.1.1 file-loader@6.2.0 -D命令
在webpack.config.js的module->rules数组中，添加loader规则

module:{
  rules:[
    {test:/\.jpg|png|gif$/,use:'url-loader?limit=229'},
  ]
}
//?之后 loader的参数值
//limit用来指定图片的大小，单位字节 kb 用于限制大图片不需要转base64
//只有<= limit大小的图片，才转为base64格式的图片

//在配置url-loader的时候，多个参数之间，使用&符号分割
要把dist文件夹删掉
module:{
  rules:[
    {test:/\.jpg|png|gif$/,use:'url-loader?limit=229&&outputPath=images'},
  ]
}

# 1.2.2 数据绑定指令

  • v-once 只编译一次
    显示内容后不再具有响应式功能

# 4.6 作用域插槽

父组件对子组件的内容进行加工处理
:info接收子组件定义中循环的item
父组件定义slotProps用于储存打印可得**{ "info": { "id": 2, "name": "c" } }**
子组件的数据通过slot-scope属性传递到了父组件

<body>
    <div id="app">
        <abc-list :list='list'>
            <template slot-scope="slotProps">
                <strong v-if="slotProps.info.id==2" class="red">{{slotProps.info.name}}</strong>
                <span v-else>{{slotProps.info.name}}</span>
            </template>
        </abc-list>
    </div>
<script type="text/javascript" src="js/vue.min.js"></script>
<script type="text/javascript">
    Vue.component('abc-list',{
        props:['list'],
        template:`
        <div>
            <li :key='item.id' v-for='item in list'>
                <slot :info='item'>{{item.name}}</slot>
            </li>
        </div>
        `
    })
