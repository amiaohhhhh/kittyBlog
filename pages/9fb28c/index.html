面向对象

# 面向对象

把事务分解成一个个对象然后由对象之间分工与合作
灵活，代码可复用，容易维护和开发的优点

# 特性

  • 封装性
  • 继承性
  • 多态性

# 1 ES6中的类和对象

# 1.1 对象

对象是一个具体的事物，看得见摸得着的事物。一本书，一辆汽车
在js中，对象是一组无序的相关属性的方法的集合，所有的事物都是对象，例如字符串、数据、数组等
对象由属性和方法

  • 属性：事物的特征
  • 方法：事物的行为

# 1.2 类 class

类抽象了对象的公共部分，泛指了某一大类
对象特指某一个，通过类实例化一个具体的对象
面向对象的思维特点

  1. 抽取(抽象)对象共用的属性和行为组织(封装)成一个类
  2. 对类进行实例化，获取类的对象

# 1.3 类语法

通过class关键字创建类，类名习惯首字母大写
类里面有constructor函数，可以接受传递过来的参数，同时返回实例对象
constructor函数只要new生成实例时，就会自动调用这个函数，如果不写也会自动生成函数
语法规范：
创建类 类名后不跟小括号，生成实例，类名后加小括号，构成函数不需要加function

//创建一个叫明星的类
class star{
	constructor(uname){
  	this.uname = uname
  }
  //创建方法
  sing(song){
  	console.log(this.uname + song)
  }
}
//利用类创建对象new
var ldh = new star('ldh')
console.log(ldh.uname)
//调用方法
ldh.sing('aaa')
console.log(ldh)//输出ldhaaa

son.sum()

# 1.5 ES6中的类和对象的注意点

在ES6中类没有变量提升，所有必须先定义类，才能通过类实例化对象
类里面的共有属性和方法一定要加this使用
注意类的指向问题

  • constructor指向实例对象
  • 方法指向的是调用的函数
  • 如果在方法中调用类中数据可以在外围定义var that方便调用

# 2.2 构造函数

构造函数是一种特殊的函数，主要是用来初始化对象，对象成员变量赋初始值，它总与new一起使用，把对象中一些公共的属性和方法抽取出来，然后封装到这个函数里面

# 2.2.1 new在执行中会做四件事情

  1. 在内存中创建一个新的空对象
  2. 让this指向这个新的对象
  3. 执行构造函数里面的代码，给这个新对象添加属性和方法
  4. 返回这个新对象(构造函数不需要return)

# 2.2.2 实例成员和静态成员

构造函数中的属性和方法称为成员，可以添加成员
实例成员：构造函数内部通过this添加成员
ldh.sing()
静态成员：通过构造函数本身上添加的成员 只能通过构造函数来访问
Star.sex = '男'

# 2.2.3 构造函数的问题

构造函数方法很好用，但是存在浪费内存的问题

# 2.2.4 构造函数原型 prototype

构造函数通过原型奉分配的函数是所有对象所共享的
每一个构造函数都有一个prototype属性，指向另一个对象，prototype是一个对象，这个对象的所有属性和方法，都会被构造函数所拥有
一般情况下，公共属性定义到构造函数里面，公共方法可以方法原型对象上

Star.prototype.sing = function(){
console.log('sss')
}

# 2.2.5 对象原型 proto

对象都会有一个属性 __proto__指向构造函数的prototype原型对象，因为__proto__的存在，对象可以使用构造函数prototype原型对象的属性和方法

  • __proto__对象原型和原型对象prototype是等价的

ldh._ proto_  === Star.prototype

  • 意义在于为对象的查找机制提供一个方向，但是是一个非标准属性，因此实际开发中，不可以使用这个属性，它只是内部指向原型对象prototype

# 2.2.6 构造函数 constructor

__proto__和prototype都有一个属性constructor属性，constructor称为构造函数，指回构造函数本身
记录对象引用哪个构造函数，它可以让原型对象重新指向原来的构造函数
如果修改了原来的原型对象，给原型对象赋值的是一个对象，则必须手动利用constructor指回原来的构造函数

Star.prototype = {
	constructor:Star,
  sing:function(){
  	console.log('aaa')
  }
}

# 3.1.4 严格模式

  1. ES5的严格模式是采用限制性JS变体的一种模式，在严格条件下运行js代码
    • 消除了js语法的一些不合理、不严谨之处，减少了一些怪异行为
    • 消除代码运行的一些不安全之处，保证代码运行的安全
    • 提高编译器效率
    • 禁用了在未来版本中可以会定义的语法，为未来新版本做铺垫
  2. 开启严格模式
//为脚本开启严格模式
<script>
  (function(){
  	'use strict';
  })();
</script>
//为某个函数开启严格模式
<script>
  function fn(){
 		 'use strict';
  }
</script>

3.严格模式的变化
this指向

  • 变量名必须先声明再使用 不能随意删除已经声明好的变量
  • 全局作用域中this是undefined
  • 以前构造函数时不加new也可以调用，当普通函数，this指向全局对象
  • 严格模式下，如果构造函数不加new调用，this会报错
  • new实例化的构造函数指向构建的对象实例
  • 定时器this还是指向window
  • 事件、对象还是指向调用者

函数变化

  • 函数名内不允许有重复的参数
  • 必须声明在顶层作用域 不允许在非函数的代码块内声明函数
